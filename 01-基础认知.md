# 01 基础认知

## 1. 什么是隐写

隐写的核心不是“加密”，而是“隐藏存在感”。

- 加密关注的是：别人看到数据，也读不懂
- 隐写关注的是：别人最好连“这里有数据”都没意识到

在 CTF 里，这两者经常叠加出现：

- 第一层是隐写：把内容藏进图片、音频、文档、流量、压缩包、文件尾部
- 第二层是加密：即使提取出载荷，还要再解码、解密、爆破口令

所以做隐写题时，不要默认“提出来就结束了”。很多题真正的坑在第二层。

## 2. CTF 中常见的隐写类型

### 2.1 文件结构类

这类题不一定改图片像素，而是利用文件格式的容错性或解析差异。

典型形式：

- `JPEG + ZIP` 文件拼接
- PNG 里插入异常 chunk
- GIF 后面挂额外数据
- 一个文件同时可被当成两种格式解析

特点：

- 最适合先做十六进制、文件头、文件尾检查
- 人眼看图通常看不出异常
- 但二进制结构里会留下强证据

### 2.2 元数据类

信息藏在：

- EXIF
- XMP
- 注释段
- ICC profile
- 文档属性

特点：

- 提取快
- 隐蔽性一般
- 新手最容易漏，因为总把注意力放在图像内容本身

### 2.3 像素域隐写

典型是 LSB。

做法是把数据塞进：

- RGB 通道最低位
- Alpha 通道
- 调色板索引

特点：

- 教学题很多
- 容量大
- 如果嵌入得粗糙，统计特征明显

### 2.4 频域隐写

常见于 JPEG。

它不直接改像素，而是改压缩过程中使用的 DCT 系数。常见思路：

- JSteg
- F5
- OutGuess

特点：

- 比简单 LSB 隐蔽
- 更像正规隐写
- 新手不容易直接看出来

### 2.5 多媒体隐写

包括：

- 音频隐写
- 视频隐写
- 帧提取
- 频谱/相位/回声隐写

特点：

- 线索可能分散在多个时间片
- 容量大
- 很多题会把二维码、文本或压缩包拆在多帧里

### 2.6 网络隐写

把信息藏进：

- DNS 查询
- HTTP 头
- 包长度
- 时间间隔
- TLS 元数据

这类题更偏取证和流量分析，不是传统图片隐写，但思维方式一致：先找异常，再还原载荷。

## 3. 做题的标准流程

建议你形成下面这套固定流程。比赛时紧张，流程越稳定越重要。

### 第一步：识别文件类型

先不要急着“跑 steghide”。

要先确认：

- 文件扩展名和真实格式是否一致
- 是单文件还是文件夹
- 是否有明显的伴随文件
- 文件大小是否可疑

你应该优先问自己：

- 这真的是一张纯图片吗
- 它会不会只是个容器
- 它后面会不会拼了别的东西

### 第二步：检查文件结构和元数据

优先排查：

- 文件头
- 文件尾
- 结束标记后是否有多余数据
- EXIF/XMP/Comment
- 是否出现 `PK`, `Rar!`, `7z`, `SQLite`, `PDF`, `flag`, `Salted__` 等关键字

### 第三步：提取所有“看起来不像图片本体”的东西

比如：

- 文件尾追加的 ZIP
- PNG 异常 chunk
- Base64 字符串
- 看起来像十六进制、二维码、压缩流的片段

这一步的目标不是立刻得 flag，而是把“隐写层”和“解密层”分离。

### 第四步：判断是不是还叠了编码或加密

常见信号：

- `U2FsdGVkX1`：OpenSSL `Salted__`
- `H4sIA`：Gzip Base64
- `eJ`：zlib 压缩的 Base64
- 一长串高熵数据：可能是加密结果
- 长度刚好对齐：可能是块加密

### 第五步：再回头检查图像/音频/视频内容本身

如果文件结构没问题，再去查：

- 通道分离
- LSB
- 色彩反相
- 对比度拉伸
- 频谱图
- 帧拆分

顺序不要反。很多人一上来就盯像素，效率很低。

## 4. CTF 隐写里最常见的组合拳

题目经常不是单点，而是两层或三层：

- 图片尾部拼 ZIP + ZIP 里有加密文本
- EXIF 给口令 + steghide 真正藏数据
- GIF 帧里有二维码 + 二维码内容再 Base64
- 音频频谱出文本 + 文本是维吉尼亚或栅栏密码

所以你要习惯问：

- 这个线索是终点，还是只是下一层的钥匙

## 5. 初学者最容易犯的错

### 错误 1：只会按工具名乱试

这会导致：

- 浪费时间
- 看不懂输出
- 对失败没有解释

正确做法：

- 先根据格式和异常选工具

### 错误 2：提取到东西就以为结束

比如提取到：

- Base64
- 十六进制
- 压缩流
- 假 flag

这通常只是中间层。

### 错误 3：忽略文件尾

文件尾追加是 CTF 非常高频的套路。因为：

- 好做题
- 好出题
- 学习价值高

### 错误 4：忘记记录证据链

正确的解题复盘应该能回答：

- 为什么你怀疑这里有隐写
- 你用什么证据确认
- 你怎么提取
- 你怎么解密
- 中间有哪些误导项

## 6. 你现在就该记住的赛时口诀

先看类型，再看结构；  
先提载荷，再谈解密；  
先抓证据，再跑工具；  
一个结果，不要默认就是终点。
