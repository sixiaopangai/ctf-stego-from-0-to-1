# 03 常用工具与排查流程

这一章最适合比赛时直接翻。

## 1. 赛时排查总流程

拿到题目文件后，优先级建议如下：

1. 确认真实文件类型
2. 检查元数据和文件结构
3. 查文件尾、可打印字符串、可疑魔数
4. 提取所有非本体载荷
5. 判断是否叠加编码/加密
6. 结构没异常，再看像素域/频域/音视频层

## 2. 最常用工具清单

### 2.1 通用取证

- `file`：识别真实格式
- `strings`：提取可打印字符串
- `xxd` / 十六进制查看器：看文件头尾和魔数
- `binwalk`：扫压缩包、固件、内嵌格式
- `foremost`：按文件头批量雕刻

### 2.2 图片相关

- `exiftool`：元数据
- `stegsolve`：图层、通道、反相、位平面
- `zsteg`：PNG/BMP LSB 检测
- `steghide`：经典隐写工具，常见于 JPG/WAV
- `outguess`：JPEG 隐写
- `pngcheck`：PNG chunk 检查
- `identify` / `magick identify`：图片属性

### 2.3 音视频相关

- `ffmpeg`：拆帧、抽音轨、转码
- `ffprobe`：查看流信息
- Audacity：频谱、相位观察

### 2.4 密码学配套

- `openssl`
- CyberChef
- 自己写的小脚本

## 3. Windows PowerShell 常用命令

### 3.1 查文件大小和基本信息

```powershell
Get-ChildItem -LiteralPath '.\题目文件'
```

### 3.2 读文件头前 64 字节

```powershell
$bytes = [System.IO.File]::ReadAllBytes('.\题目.jpg')
($bytes[0..63] | ForEach-Object { '{0:X2}' -f $_ }) -join ' '
```

### 3.3 找关键字

```powershell
$latin1 = [System.Text.Encoding]::GetEncoding('ISO-8859-1').GetString($bytes)
@('PK','Salted__','flag','Exif','JFIF','PNG','GIF89a') | ForEach-Object {
    "$_ => " + $latin1.IndexOf($_)
}
```

### 3.4 查 JPEG 结束后是否有尾巴

```powershell
$lastEOI = -1
for ($i = 0; $i -lt $bytes.Length - 1; $i++) {
    if ($bytes[$i] -eq 0xFF -and $bytes[$i + 1] -eq 0xD9) {
        $lastEOI = $i
    }
}

"Trailing bytes: " + ($bytes.Length - ($lastEOI + 2))
```

### 3.5 直接把尾巴存下来

```powershell
$tail = $bytes[($lastEOI + 2)..($bytes.Length - 1)]
[System.IO.File]::WriteAllBytes('.\tail.bin', $tail)
```

## 4. 图片题的决策树

### 情况 A：文件尾有额外数据

优先怀疑：

- ZIP/RAR/7z 拼接
- 二进制载荷
- 自定义密文

动作：

- 提取尾巴
- 识别魔数
- 尝试按压缩包打开

### 情况 B：有 EXIF/XMP/Comment

优先怀疑：

- 口令藏在元数据
- 直接藏了 Base64
- 给出下一层提示

动作：

- 完整导出元数据
- 看拍摄软件、作者、注释、用户字段

### 情况 C：结构正常，但 PNG/BMP

优先怀疑：

- LSB
- Alpha
- 调色板

动作：

- `zsteg`
- `stegsolve`
- 通道分离

### 情况 D：结构正常，但 JPEG

优先怀疑：

- steghide
- outguess
- F5 / JSteg 一类频域隐写

动作：

- 先试口令线索
- 再试 JPEG 专项工具

### 情况 E：音频/视频

优先怀疑：

- 频谱文本
- 拆帧二维码
- 低音轨/高音轨分层
- 元数据或字幕轨

动作：

- `ffprobe`
- `ffmpeg -i`
- 抽帧
- 频谱观察

## 5. `strings` 类思维非常重要

即使没有 `strings` 命令，你也应该有同样的意识：

- 文件里有没有可打印文本
- 有没有 Base64
- 有没有压缩包标记
- 有没有盐值、算法头、JSON、XML、URL

这一步极其高性价比。

## 6. 比赛时最该先试哪些口令

如果工具提示需要 password，不要乱爆破。先从题面和文件里找：

- 文件名
- 题目名
- 作者名
- 假 flag
- EXIF 注释
- 图片上可见文本
- 附件名
- 同目录其他文件名

CTF 里口令通常不是完全随机，而是“能被你推出来”的。

## 7. 什么时候该自己写脚本

满足下面任意一个条件，就别纠结 GUI 了，直接写脚本：

- 需要重复试多组口令
- 需要批量扫多个文件
- 需要解析简单固定格式
- 工具输出太多，你只想提取其中一个字段
- 你已经确认原理，只差自动化

## 8. 常见误判

### 误判 1：看到假 flag 就停

正确做法：

- 看它是不是诱饵
- 看是否还有更高熵的东西

### 误判 2：所有图片题都先跑 LSB

正确做法：

- 先按格式分类
- JPEG 先结构和频域
- PNG/BMP 再优先像素域

### 误判 3：把工具报错当“没有隐写”

很多报错只是：

- 口令不对
- 格式不匹配
- 文件损坏
- 工具不支持该方法

## 9. 一份赛时 checklist

下面这份可以直接照着过一遍：

```text
[ ] 文件扩展名和真实格式一致吗
[ ] 文件头尾正常吗
[ ] 结束标记后有多余数据吗
[ ] 有 EXIF/XMP/Comment 吗
[ ] 有明显字符串、Base64、Salted__、PK 吗
[ ] 能提取出额外压缩包或文本吗
[ ] 结果是终点还是下一层线索
[ ] 是否需要口令，口令线索来自哪里
[ ] PNG/BMP 是否查过 LSB/Alpha/Palette
[ ] JPEG 是否考虑过 steghide/outguess/F5
[ ] 音视频是否拆帧/看频谱
```

## 10. 这一章的核心目标

把工具变成流程的一部分，而不是碰运气的按钮。
