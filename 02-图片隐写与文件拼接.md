# 02 图片隐写与文件拼接

这一章只讲 CTF 里最常见、最值得优先检查的一类：图片文件相关隐写。

## 1. 先理解为什么“图片后面拼 ZIP”成立

以 JPEG 为例：

- JPEG 以 `FFD8` 开头
- JPEG 以 `FFD9` 结束

对大多数图片查看器来说，只要读到 `FFD9`，图片就结束了。  
如果有人在 `FFD9` 之后再追加一段 ZIP 数据：

- 图片依然能正常显示
- 但整个文件从二进制角度看已经变成“图片 + 额外载荷”

这就是最经典的文件拼接隐写。

## 2. JPEG 里最值得先看的结构

常见标记：

- `FFD8`：SOI，开始
- `FFE0`：APP0，常见 JFIF
- `FFE1`：APP1，常见 EXIF
- `FFDA`：SOS，扫描数据开始
- `FFD9`：EOI，结束

赛时实用结论：

- 重点看 `FFD9` 后有没有额外字节
- 重点看 APP 段里有没有 EXIF/XMP/Comment

## 3. JPEG+ZIP 的实战排查

### 3.1 思路

排查顺序：

1. 找到 JPEG 结束位置
2. 计算结束后还有没有尾随数据
3. 看尾随数据是不是 `PK 03 04`
4. 如果是，就按 ZIP 解

### 3.2 PowerShell 示例

```powershell
$path = '题目.jpg'
$bytes = [System.IO.File]::ReadAllBytes($path)

$lastEOI = -1
for ($i = 0; $i -lt $bytes.Length - 1; $i++) {
    if ($bytes[$i] -eq 0xFF -and $bytes[$i + 1] -eq 0xD9) {
        $lastEOI = $i
    }
}

"EOI 位置: $lastEOI"
"尾随字节数: " + ($bytes.Length - ($lastEOI + 2))
```

如果尾随字节数大于 0，再看尾巴：

```powershell
$tail = $bytes[($lastEOI + 2)..($bytes.Length - 1)]
($tail[0..15] | ForEach-Object { '{0:X2}' -f $_ }) -join ' '
```

如果开头是：

```text
50 4B 03 04
```

那就是 ZIP。

### 3.3 直接提取 ZIP 尾巴

```powershell
$zipBytes = $bytes[($lastEOI + 2)..($bytes.Length - 1)]
[System.IO.File]::WriteAllBytes('hidden.zip', $zipBytes)
```

然后你就可以正常解压它。

## 4. 这类题的典型第二层：载荷再加密

很多题不会把真 flag 直接塞进 ZIP，而是放一层诱饵。

例如：

```text
flag{this_flag_is_a_lie}
U2FsdGVkX1/zdL/aAU73J9nRW9GWY2nOOHefWO0JiUnr3kMEcXUtXNS0bOeDTGOM
```

第一行看起来像 flag，但题目明确暗示它是假的。  
第二行才是关键，它是一个 Base64 字符串。

## 5. 如何识别 OpenSSL `Salted__` 密文

看到下面这种 Base64 前缀时要立刻警觉：

```text
U2FsdGVkX1
```

它 Base64 解码后通常以：

```text
53 61 6C 74 65 64 5F 5F
```

也就是：

```text
Salted__
```

这说明它很可能是 OpenSSL `enc` 生成的加密结果。

常见参数组合：

- `AES-128-CBC`
- `AES-192-CBC`
- `AES-256-CBC`
- `-salt`
- 旧题中常见 `MD5` 做 EVP_BytesToKey 派生

## 6. 典型解题套路

假设你已经从图片尾部提取出 `flag.txt`，内容如上。

这时你的正确思路应该是：

1. 不要把第一行假 flag 当终点
2. 先判断第二行是什么格式
3. 发现它是 `Salted__`
4. 再思考第一行是不是口令
5. 用第一行作为 password 去解第二行

这就是非常典型的：

- 隐写层：文件拼接
- 解密层：OpenSSL 密文

## 7. 对应的 OpenSSL 命令

如果系统里有 `openssl`，可以直接试：

```powershell
$cipher = 'U2FsdGVkX1/zdL/aAU73J9nRW9GWY2nOOHefWO0JiUnr3kMEcXUtXNS0bOeDTGOM'
$cipher | openssl enc -aes-256-cbc -d -a -salt -md md5 -pass pass:"flag{this_flag_is_a_lie}"
```

如果参数猜对，就会输出明文。

## 8. 不装 OpenSSL 也能解

本教程附带了：

- `scripts/Decrypt-OpenSslSalted.ps1`

用途：

- 输入 Base64 密文
- 输入口令
- 按 OpenSSL 常见的 `AES-CBC + MD5 EVP_BytesToKey` 方式尝试解密

## 9. 图片内容本身还要不要查

要，但优先级在文件结构之后。

合理顺序是：

1. 先看结构
2. 再提尾巴
3. 再查元数据
4. 最后才看通道、反相、LSB、对比度拉伸

因为在很多入门到中级题里，文件结构异常的命中率非常高。

## 10. 图片内容隐写的基础入口

如果结构和元数据都没异常，再考虑这些方向：

- 通道分离：R/G/B
- 反相、拉高对比度
- 查看 Alpha
- LSB bit plane
- PNG 调色板顺序
- JPEG DCT 隐写

### 10.1 通道分离

目的：

- 看文字是否只藏在某一个颜色通道里
- 看是否有局部涂层或低透明度覆盖

### 10.2 LSB

最低位隐写最常见的特征：

- 正常图像基本看不出来
- 但 bit plane 展开后会出现纹理异常、图案、二维码、文字边缘

### 10.3 JPEG 频域

如果文件是 JPEG，不要执着于 LSB。  
因为 JPEG 是有损压缩，很多像素级方法并不稳定。此时更该考虑：

- JSteg
- OutGuess
- F5

## 11. 这一类题的赛时最小流程

拿到一张图后，先按下面顺序走：

1. 看文件头尾是否正常
2. 看 `FFD9` 后有没有数据
3. 看尾巴是否是压缩包或其他文件头
4. 查 EXIF/XMP/Comment
5. 查是否出现 `flag`, `Salted__`, `PK`, `Base64`
6. 没收获再做通道、LSB、频域

## 12. 这一章的核心结论

很多所谓“图片隐写题”，本质上根本不是像素隐写，而是：

- 图片作为伪装外壳
- 载荷藏在文件结构外围
- 真正难点落在后续编码或加密

所以你做题时一定要分清：

- 你是在找“东西藏哪了”
- 还是在解“拿出来以后它是什么”
