# 06 音频与视频隐写

这一章的目标不是把多媒体处理讲成数字信号处理课程，而是让你遇到音频和视频隐写题时，知道应该先看什么、后看什么。

## 1. 为什么音视频隐写适合出题

和单张图片相比，音视频的冗余更高，藏数据的方法也更多：

- 时间轴可以切分
- 左右声道可以分离
- 频谱可以画图
- 视频可以拆成大量帧
- 一道题可以天然分成多层线索

这意味着：

- 信息可以藏在某一个时刻
- 也可以分散到整段时间里
- 还可以把“提取”和“拼接”设计成两步题

## 2. 音频题先看什么

拿到音频文件，建议先做这四步：

1. 看基础信息：时长、采样率、声道数、编码格式
2. 分析波形和频谱
3. 检查元数据和可打印字符串
4. 再考虑 LSB、相位、回声、分段拼接

### 2.1 基础信息

先搞清楚：

- 是 `wav`、`mp3`、`flac` 还是别的格式
- 是单声道还是双声道
- 时长是否过长或过短
- 码率和采样率是否异常

你真正想知道的是：

- 这题是“声音内容题”
- 还是“文件结构题”

## 3. 音频题最常见的几类

### 3.1 频谱出文字

这是 CTF 里最经典的音频题之一。

做法通常是：

- 把文本、二维码、图案画在频谱上
- 肉耳未必明显异常
- 但在频谱视图里一眼能看到

赛时动作：

- 用 Audacity 或其他工具打开频谱视图
- 调整窗口大小、FFT size、对数坐标
- 看高频和低频有没有文字或轮廓

### 3.2 左右声道差分

信息可能藏在：

- 左声道单独有内容
- 右声道单独有内容
- 两个声道相减后出现文本或节奏

所以音频题不要只听“混合后的总效果”，要分开看左右声道。

### 3.3 LSB 音频隐写

常见于未压缩音频，尤其是 `wav`。

特点：

- 原理和图片 LSB 类似
- 常需要专门工具或脚本提取

### 3.4 反向播放/变速/静音段

很多音频题并不复杂，只是把线索藏在：

- 反放
- 慢放
- 提速
- 静音段
- 某个非常短的片段

这类题比算法题更像“观察题”，但比赛里非常高频。

## 4. 音频题的实战流程

推荐顺序：

1. `ffprobe` 看流信息
2. 查元数据
3. 看波形
4. 看频谱
5. 拆左右声道
6. 导出若干关键时间片
7. 最后再怀疑高级隐写

## 5. 视频题先看什么

视频题的关键是分层分析。

你至少要把它拆成三层看：

- 视频画面
- 音频轨
- 容器和字幕/附件流

很多新手只盯着画面，这是错误的。  
正确思路是先问：

- 线索在画面里吗
- 在音轨里吗
- 在字幕或元数据里吗
- 还是在帧序列变化里

## 6. 视频题最常见的几类

### 6.1 拆帧题

典型特征：

- 肉眼看视频正常
- 但逐帧拆开后能看到：
- 文字
- 二维码
- 多帧拼图
- 定时闪烁内容

赛时动作：

- 抽帧
- 低频采样浏览
- 再重点看可疑时间点

### 6.2 每隔 N 帧取一张

有些题目会把信息拆到：

- 奇数帧
- 偶数帧
- 每 5 帧
- 每秒一帧

这类题如果只正常播放，很容易错过。

### 6.3 视频里其实藏的是音频题

很多视频题真正的核心在线索音轨里：

- 频谱图
- 摩斯码
- DTMF
- 左右声道差异

所以视频题也经常需要先把音频抽出来单独分析。

### 6.4 字幕轨/附件流

容器格式里可能还带：

- 字幕
- 附件
- 章节
- 注释

如果你只看播放画面，会完全漏掉这层。

## 7. `ffmpeg/ffprobe` 最值得会的几类操作

### 7.1 看流信息

```powershell
ffprobe 题目.mp4
```

重点看：

- video/audio/subtitle 是否都有
- 时长
- 编码器
- 分辨率
- 比特率

### 7.2 抽音轨

```powershell
ffmpeg -i 题目.mp4 -vn audio.wav
```

### 7.3 拆帧

```powershell
ffmpeg -i 题目.mp4 frames/frame_%05d.png
```

### 7.4 每秒抽一帧

```powershell
ffmpeg -i 题目.mp4 -vf fps=1 frames/frame_%05d.png
```

## 8. 做音视频题时的常见信号

### 信号 1：时长明显异常

比如：

- 文件很大但时长很短
- 文件很小但时长很长

这通常说明：

- 码率异常
- 有隐藏流
- 数据结构可疑

### 信号 2：波形很奇怪

例如：

- 某段几乎静音但其实有信息
- 左右声道差异巨大
- 波形整齐得像编码数据

### 信号 3：画面有周期性闪烁

这通常意味着：

- 有帧间藏图
- 有交替帧
- 有时间维隐写

## 9. 音视频题的误区

### 误区 1：只靠耳朵听

很多题的关键信息不是“听出来”，而是“看出来”：

- 频谱
- 波形
- 声道差分

### 误区 2：只看正常播放效果

很多视频题必须拆帧才能看见。

### 误区 3：不会分层

正确拆法应该是：

- 容器层
- 流层
- 时间层
- 内容层

## 10. 这章的最小赛时 checklist

```text
[ ] 看过流信息了吗
[ ] 有几条音轨/字幕轨
[ ] 把音频单独抽出来了吗
[ ] 看过频谱了吗
[ ] 左右声道分开看了吗
[ ] 抽帧看过了吗
[ ] 有没有按时间间隔采样
[ ] 有没有可疑闪烁或隐藏二维码
[ ] 是否检查过元数据和字符串
```

## 11. 这一章的核心结论

音视频隐写并不一定比图片更“高级”，但它更依赖分层思维。  
你一旦养成“容器、流、时间轴、内容”四层拆解习惯，很多原本看起来复杂的题会立刻清晰很多。
